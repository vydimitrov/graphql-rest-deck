Is GraphQL The Better Alternative of REST

What are REST and GrphQL

REST (Representational State Transfer)
GraphQL (Graph Query Language)
  • two ways to transfer data between the client and server
  • REST is an API design architecture whereas GraphQL is a query language/sification

REST Overview

Everything is a resource which is identified by a URL

/api/photo/list
/api/live/list
Each request returns the whole resource object

REST Architecture

Multiple requests to fetch data for different resource

REST Limitations

  • Over fetching (99% of the requests)
  • Multiple round trips to fetch related resources
  • Very hard to maitain good documnetation

GraphQL Overview

  • Single endpoint
  • All request are executed against a schema
  • The schema is strongly typed
  • GraphQL is language agnostic
  • Three operations - quering, mutating, and subscription

GraphQL Client-Server

The server returns only the data the client requests - in very flexaible and predictable way
